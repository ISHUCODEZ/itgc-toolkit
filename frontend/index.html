IT GENERAL CONTROLS · TESTING TOOLKIT

Control health, right now.

Live monitoring across all four ITGC pillars — access management, change management, operations & monitoring, and JML access revocation. Each tool runs the test an auditor would, writes exceptions as documented findings, and feeds the continuous monitoring layer that trends control health over time.

PILLAR 1 · ACCESS · SOD
SoD Analyzer
Upload a user-entitlement export. Flags toxic role combinations — e.g. create-vendor + approve-payment — against an explicit conflict ruleset.
PILLAR 1 · ACCESS · JML
JML Access Review
Cross-references terminated employees against the live entitlements population. Every active role held by a former employee is a documented finding.
PILLAR 2 · CHANGE
Change Auditor
Reads a change-ticket log and tests each change for approval, test evidence, and developer-vs-deployer segregation.
PILLAR 3 · OPERATIONS
Recertification
Tracks the periodic access-review cycle — shows which grants are current, due soon, overdue, or never reviewed.
MONITORING LAYER
CCM Monitor
Trend control health over time. Each run is a timestamped snapshot; findings carry stable fingerprints for aging, delta detection, and MTTR.
CROSS-CUTTING
Framework Map
Crosswalks one control across SOX ITGC, COBIT, ISO 27001, NIST CSF and SOC 2 — show one control satisfies many frameworks.

What this demonstrates

The four ITGC access controls — SoD (who holds toxic combinations), JML (terminated users still holding access), change management (changes reaching production under control), and recertification (access re-reviewed on cadence) — plus continuous monitoring and the framework crosswalk. Every finding traces to a named rule with a risk rationale and a control reference, exactly how a real audit exception is documented. The toolkit also practises the controls it tests: role-based access, a full activity audit trail, and exception sign-off workflow.
