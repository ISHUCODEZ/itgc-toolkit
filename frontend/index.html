Test the four pillars of ITGC — with evidence.

A working toolkit for the controls a Technology Risk team tests every engagement: access management, change management, computer operations, and the control frameworks that tie them together. Upload a real export — entitlements, change tickets, access reviews — and each tool runs the test an auditor would and writes the exceptions as documented findings.

PILLAR 1 · ACCESS
SoD Analyzer
Upload a user-entitlement export. Flags toxic role combinations — e.g. create-vendor + approve-payment — against an explicit conflict ruleset.
PILLAR 2 · CHANGE
Change Auditor
Reads a change-ticket log and tests each change for approval, test evidence, and developer-vs-deployer segregation.
PILLAR 3 · OPERATIONS
Recertification
Tracks the periodic access-review cycle — shows which grants are current, due soon, overdue, or never reviewed.
CROSS-CUTTING
Framework Map
Crosswalks one control across SOX ITGC, COBIT, ISO 27001, NIST CSF and SOC 2 — show one control satisfies many frameworks.

What this demonstrates

The three core ITGC domains — access management (who can do what, and is it still appropriate), change management (did changes reach production under control), and operations & monitoring (is access re-reviewed on cadence) — plus the framework crosswalk auditors use to map a single control across standards. Every tool is deterministic and rule-based: each finding traces to a named rule with a risk rationale and a control reference, exactly how a real audit exception is documented. The toolkit also practises the controls it tests — role-based access and a full activity audit trail.
